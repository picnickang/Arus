1. EXECUTIVE SUMMARY
────────────────────
This upgrade transforms the ARUS PdM system from a basic ML prediction pipeline
into an industrial-grade platform with five new backend domains, seven new
database tables, five frontend hooks, and a unified PdM Platform UI page.

Key capabilities added:
  • Feature Store — Statistical feature extraction from equipment telemetry
  • Fleet Analytics — Fleet-wide baselines and equipment comparison via z-scores
  • Model Registry — Full model lifecycle management with versioning & deployment
  • Inference Pipeline — End-to-end prediction with latency tracking & explanations
  • Model Monitoring — Drift detection comparing live distributions to training data

7. NEW SERVICE ARCHITECTURE
───────────────────────────
Each pdm-platform domain follows hexagonal architecture:

  ports.ts    → Domain interface definitions
  adapter.ts  → Infrastructure adapter (Drizzle/PostgreSQL)
  routes.ts   → HTTP interface (Express router)

Data Flow:
  Equipment → Telemetry → Feature Store (compute) → Features
  Features → Fleet Analytics (compare) → Z-Scores / Baselines
  Features → Inference Pipeline (run) → Predictions + Explanations
  Predictions → Model Monitoring (drift) → Drift Metrics

  Model Registry manages: Models → Versions → Deployments
  Inference Pipeline resolves: Equipment → Active Deployment → Model Version → Features → Prediction

8. REMAINING STUBS
──────────────────
  • Feature Store compute: Returns deterministic stub features (no real telemetry
    aggregation yet — requires telemetry data in TimescaleDB to be meaningful)
  • Inference stub-runner: Returns deterministic predictions based on feature
    heuristics (temperature/vibration thresholds) — real model runtime TBD
  • Fleet baselines compute: Aggregates from equipment_features table (needs
    populated feature data to produce meaningful baselines)
  • Drift compute: Returns stub drift scores — real KL-divergence/PSI computation
    requires accumulated live inference data vs training distributions

  All stubs are clearly marked and produce plausible, deterministic output.
