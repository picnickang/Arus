Perform a safe, true semantic TypeScript cleanup of ARUS until `npm run typecheck` passes with 0 errors.

Hard constraints:
- Do NOT use `// @ts-nocheck`.
- Do NOT disable `strict`.
- Do NOT weaken tsconfig.
- Do NOT mass-cast to `any`.
- Do NOT revert LLM gateway, OpenAI gateway migration, crew hexagonal DI, repository boundaries, or domain architecture.
- Preserve existing runtime behavior unless a type error exposes a real bug.
- Keep changes small, semantic, and reviewable.
- Run `npm run typecheck` after each cluster and continue until 0 errors.
- Run `npm run check:guards` before finishing.
- Regenerate `ts-burndown-baseline.json` only after the compiler is clean or after a verified semantic reduction.

Cleanup strategy:
1. Start with missing imports/exports:
   - Fix TS2307 and TS2305 first.
   - Prefer barrel exports over duplicate types.
   - Do not invent parallel enums if a canonical type already exists.

2. Fix API response typing:
   - Add generics to API helper calls.
   - Replace `unknown` result usage with explicit DTO interfaces.
   - Do not cast raw API responses directly into domain entities unless the API contract already guarantees that shape.

3. Separate model layers:
   - DomainEntity = database/domain truth.
   - ApiDto = API response/request contract.
   - UiViewModel = component rendering shape.
   - FormValues = react-hook-form shape.
   Add mapper functions where UI expects denormalized fields.

4. Fix forms semantically:
   - Form values should use strings, numbers, booleans, undefined.
   - Do not put DB `null` directly into controlled inputs.
   - Normalize DB/API values into form defaults.
   - Normalize form values back into API payloads.
   - Use one consistent `useForm<FormValues>()` generic per form.

5. Fix nullability:
   - For React input values, use `field.value ?? ""`.
   - For optional dates, format only after guarding.
   - For nullable IDs, convert `null` to `undefined` in frontend form/view models.

6. Fix frontend schema drift:
   - Where UI expects fields like `equipmentName`, `completedAt`, `partName`, `partNumber`, or `quantityUsed`, create view-model mappers instead of polluting core domain types.
   - If backend truly no longer returns a field, adjust UI to use the current field or derive the display value safely.

7. Fix analytics/reporting DTO drift:
   - Create explicit analytics DTO/view-model types under the relevant feature folder.
   - Avoid changing shared domain entities just to satisfy analytics screens.

8. Fix chart and payload typing:
   - Guard possibly undefined Recharts payloads.
   - Convert enriched chart rows to the exact chart data shape expected by child components.

9. Fix server/domain errors:
   - Prefer canonical repositories from `server/repositories.ts`.
   - Preserve ports/adapters.
   - For Drizzle types, align insert/select payloads with schema definitions.
   - Do not bypass schema errors with broad casts.

10. Final verification:
   - Run `npm run typecheck`.
   - Run `npm run check:guards`.
   - Run relevant unit/integration tests for touched domains if available.
   - Provide a final summary listing:
     - starting error count
     - final error count
     - files changed
     - major clusters fixed
     - any intentionally deferred runtime/schema questions

Goal:
A production-safe TypeScript-zero state with architecture preserved, not a superficial compile pass.
