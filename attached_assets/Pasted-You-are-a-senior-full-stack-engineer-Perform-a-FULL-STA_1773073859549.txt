CONTEXT FROM REPO REVIEW
==================================================

The repo currently has an inconsistent installation/runtime story:

- Electron is still the real integrated desktop runtime
  - electron-builder.json
  - vite.config.electron.ts
  - client/src/lib/electron.ts
  - Electron-focused docs/scripts/verification/build flow

- Tauri exists but is currently only a minimal scaffold
  - src-tauri/main.rs
  - src-tauri/lib.rs
  - tauri.conf.json
  - only basic greet command / shell plugin

- Capacitor config exists for mobile

- Installer/build docs are still Electron-centered

- install-mac.sh is a developer bootstrap script, not a real customer installer
  - requires Node on target machine
  - runs npm install
  - rebuilds native modules
  - launches via npx electron
  - contains leftover naming bugs

- SQLite embedded bootstrap still seeds a default tenant identity for local mode

This means the installation and desktop runtime architecture is currently not coherent and must be consolidated.
