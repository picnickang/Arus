3. MODBUS AUTO-DISCOVERY & REGISTER MAPPING

We don’t have the official register map, so we need an auto-discovery helper that the engineer can run when the vessel is online:

Create DB tables:

telemetry_register_map (
  id                integer primary key,
  source            text,          -- 'FMCC_RMS12D4A'
  register_address  integer,
  function_code     integer,
  data_type         text,          -- 'float32', 'int32', 'uint16', 'bool', 'bitmask'
  label             text,          -- optional human label once known
  unit              text,
  confidence        real,          -- 0–1
  min_value         real,
  max_value         real,
  last_value        real,
  last_seen_at      timestamp
);

telemetry_register_mapping_metadata (
  id                integer primary key,
  created_at        timestamp,
  updated_at        timestamp,
  fmcc_host         text,
  scan_range_start  integer,
  scan_range_end    integer,
  scan_notes        text
);

FmccRegisterScanner
	•	Manually triggered (not always running).
	•	For a given range (e.g. 0–9999), periodically reads blocks of registers and tries different interpretations:
	•	float32, int32, uint16, bool, bitmask
	•	For each potential register:
	•	Measure variance, rate of change, typical range, and correlation with other registers.
	•	Try to infer likely signals:
	•	RPM: stable around 0–1500, correlated with fuel flow.
	•	Engine load: 0–110%.
	•	GPS lat/lon: values in valid ranges, change smoothly with vessel movement.
	•	SOG/COG: 0–20 knots, 0–360 degrees.
	•	FO/DO flow: correlated with RPM/load and operation mode.
	•	ROB: slowly changing totals.
	•	Store candidates in telemetry_register_map with a confidence score.

Also create/update a JSON mapping file:

/server/telemetry/adapters/fmcc-rms12d4a/fmcc-register-map.json

Structure:

{
  "meta": {
    "version": 1,
    "source": "FMCC_RMS12D4A"
  },
  "signals": {
    "gps.lat": { "address": 1200, "type": "float32", "unit": "deg" },
    "gps.lon": { "address": 1202, "type": "float32", "unit": "deg" },
    "nav.sog": { "address": 1300, "type": "float32", "unit": "kn" },
    "nav.cog": { "address": 1302, "type": "float32", "unit": "deg" },
    "engine.me.rpm": { "address": 2000, "type": "float32", "unit": "rpm" },
    "engine.me.load_pct": { "address": 2002, "type": "float32", "unit": "%" },
    "fuel.fo.flow_total": { "address": 2100, "type": "float32", "unit": "kg/h" },
    "fuel.fo.temp": { "address": 2102, "type": "float32", "unit": "°C" },
    "alarm.bitfield1": { "address": 3000, "type": "bitmask", "unit": "" }
  }
}

The auto-discovery UI (below) should let the engineer promote candidates into this JSON and into higher-confidence mappings.
