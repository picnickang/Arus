Implement a robust system that can send email alerts tied to crew and compliance, including:



1. Crew certificate expiry alerts

Already expired

Optionally Master or vessel-specific recipients

Which certificate types are tracked

When to alert (days before expiry)

Whether to send reminders (e.g. repeat notifications every X days until resolved)



2. Hours-of-rest violations alerts

Integrate with the existing hours-of-rest / crew scheduler logic.

Detect:

MLC / STCW rest non-compliance (e.g. <10h rest in 24h, <77h rest in 7 days, or other rule set you find in the existing code)

Consecutive violations for a given crew member

Create alerts when:

A violation is detected or predicted (if there is forecast logic)

There are repeated violations for the same person in a given period

Send email alerts to:

Master, DPA, HSE/compliance, crewing manager (configurable per org/vessel)

Include in emails:

Crew name, rank, vessel, date/time windows

Which rule was violated

Severity (configurable mapping)



3. Missing signatures / log completion alerts (crew side)

If the hours-of-rest forms, crew logs, or daily reports require crew signatures:

Detect unsigned logs after a configurable time window (e.g. 24h after period end).

Send reminders:

To the responsible officer / Master

Optionally to shore-side managers

Reuse the same pattern as engine log “unsigned log” notifications where applicable, but scoped to crew and rest logs.



4. Minimum manning / roster issues

Use existing crew roster + scheduler data to detect:

Watch or shift with insufficient manning vs configured minimums for that vessel.

When detected:

Generate an internal alert and optional email to Master / crewing / HSE.

Allow toggling this feature per vessel/org.



5. Crew change / crew planning reminders (basic)

If the database includes crew contract end-dates or planned crew changes:

Send reminder emails X days before planned crew off-signing, with list of affected crew and their roles.

Minimal implementation is fine, but integrate with existing models.
