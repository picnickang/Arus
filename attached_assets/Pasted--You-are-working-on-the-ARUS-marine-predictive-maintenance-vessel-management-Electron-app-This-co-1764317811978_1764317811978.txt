2️⃣ Backend: Logbook Services and APIs


Implement robust backend endpoints in server/index.ts (or appropriate router files) that:

2.1 Ensure / Get Daily Log for a Vessel

GET /api/logbook/days?vesselId=...&from=...&to=...
Lists logbook days for a vessel in a date range.
POST /api/logbook/days/ensure
Body: { orgId?, vesselId, date }
Behaviour:
If a deck_log_day for (orgId, vesselId, date) exists → return it.
Else → create it (status "open") and return.

2.2 Events Management

GET /api/logbook/days/:dayId/events
Returns all events for that day (ordered by timestamp).
POST /api/logbook/days/:dayId/events
Create a manual or system event.
Accepts all the contextual fields above.
Optional: PATCH /api/logbook/events/:id for corrections (if allowed before the day is locked).

2.3 Locking / Compliance

POST /api/logbook/days/:dayId/lock
Sets status = "locked", lockedAt, lockedByUserId.
Prevents further modification except explicitly allowed actions (e.g., append-only corrections or audit trail).

Important:
Integrate requireOrgId middleware and enforce access control consistent with the rest of the app.
Enforce that vesselId is required and correctly bound to the authenticated org.
For any write operation, ensure handling for offline embedded mode (SQLite only) still works.
