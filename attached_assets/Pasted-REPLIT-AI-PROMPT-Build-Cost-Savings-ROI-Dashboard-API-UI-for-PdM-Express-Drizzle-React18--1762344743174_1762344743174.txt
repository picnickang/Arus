2) API Layer (new routes, Zod validation, RBAC)



Create a new router routes/cost-savings.ts and mount at /api/cost-savings. Each endpoint:

Uses org-id header (string, required).

Validates query/body with zod.

Emits Prometheus metrics (counter + histogram per route).

Returns consistent { ok: true, data } or { ok: false, error }.



Endpoints (all GET):

/summary?from=<ISO>&to=<ISO>

Returns the existing getSavingsSummary(orgId, from, to).

/trend?months=<int=12>

Returns monthly series from getMonthlySavingsTrend(orgId, months).

/breakdown?from=<ISO>&to=<ISO>&groupBy=vessel|equipment|maintenanceType|triggeredBy

Aggregates totals & counts grouped by the selected dimension; include weightedSavings.

/top?limit=<int=10>&from=<ISO>&to=<ISO>&by=savings|downtime

Top N work orders/equipment by selected metric. Include equipment name, vessel name if available.

/roi?from=<ISO>&to=<ISO>

Returns:

totalSavings, weightedSavingsTotal

totalPdMSpend (sum of work order actual costs where predictive/preventive)

roiMultiple = totalSavings / GREATEST(totalPdMSpend, 1)

paybackDays (if avgSavingsPerIncident > 0, estimate payback vs. monthly PdM spend setting on org).



Input validation defaults:

If from/to missing → last 90 days.

If months missing → 12.
