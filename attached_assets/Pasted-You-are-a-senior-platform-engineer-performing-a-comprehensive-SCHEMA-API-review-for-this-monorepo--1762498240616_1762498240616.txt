# 1) Schema Audit (Postgres + SQLite vessel mode)
Create `tools/schema-audit.ts` that loads all Drizzle table definitions:

Scope:
- Enumerate **every table** and field; detect:
- Missing `orgId` on multi-tenant entities; inconsistent NOT NULL on `orgId`.
- Missing **primary keys** or composite PKs where appropriate.
- Missing or weak **foreign keys**, missing ON UPDATE/DELETE policies.
- Inconsistent **timestamps** (`createdAt`, `updatedAt` types/defaults).
- Enum/string constants drift between code & DB.
- Orphan rows (FK targets missing) with sample counts.
- Duplicate active rows that should be unique (e.g., multiple active thresholds per device/sensor).
- Missing **unique indexes** (e.g., (orgId, name) for vessels; (orgId, sku) for parts).
- Missing **performance indexes** for hot filters/joins seen in code (e.g., (orgId, equipmentId), (orgId, deviceId, sensorType), (orgId, updatedAt DESC)).
- Mis-typed numerics (cost/quantity/rates) and unit fields; threshold ordering (warnLo < critLo < warnHi < critHi).
- Parity check: **SQLite mirror** tables (e.g., `*Sqlite`) must match the canonical Postgres shapes (types mapped, JSONв†’TEXT, booleansв†’integer mode). Report all diffs.

Outputs:
- Auto-build a **Mermaid ERD** and write to `reports/erd.md`.
- Write a machine-readable diff to `reports/schema-diff.json`.
- If critical gaps are found (e.g., missing PK/FK), generate **idempotent migrations** in `db/migrations/auto/*` and print what they fix.

Run:
- `pnpm tsx tools/schema-audit.ts`
