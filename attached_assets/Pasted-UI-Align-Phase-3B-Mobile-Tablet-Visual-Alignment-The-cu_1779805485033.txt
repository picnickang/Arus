UI Align Phase 3B — Mobile/Tablet Visual Alignment

The current implementation is still too desktop-oriented and does NOT visually match the mobile/tablet reference previews closely enough.

This phase focuses ONLY on:
- responsive/mobile/tablet alignment
- operational shell behavior
- touch-first layouts
- preview parity

Do NOT add new backend logic.
Do NOT redesign architecture.
Do NOT add new modules.

==================================================
PRIMARY GOAL
==================================================

Make the mobile and tablet experience visually resemble the attached preview images much more closely.

Treat the images as:
- layout specifications
- spacing specifications
- hierarchy specifications
NOT merely inspiration.

==================================================
MOBILE/TABLET REQUIREMENTS
==================================================

The mobile UI should feel like:
- a modern maritime operational app
- a high-end fleet operations mobile console
- calm and touch-optimized

NOT:
- a shrunk desktop dashboard
- a generic admin panel
- a responsive afterthought

==================================================
REQUIRED MOBILE CHANGES
==================================================

1. Full-screen dark shell
- edge-to-edge dark background
- proper safe-area padding
- modern mobile spacing
- no floating white containers

2. Mobile operational header
- compact vessel/user context
- status pill
- notifications/alerts icon
- clear hierarchy

3. Stacked operational cards
Cards should stack vertically like the preview:
- vessel card
- shift card
- tasks card
- alerts card
- maintenance card
- feedback/report CTA

Cards should:
- have stronger visual hierarchy
- larger touch targets
- tighter operational spacing
- compact but readable typography

4. Bottom navigation
- modern floating/tab style
- touch-friendly
- compact labels/icons
- consistent with preview

5. Admin mobile/tablet command center
Tablet/mobile admin view should:
- collapse sidebar intelligently
- use stacked KPI cards
- use horizontal scroll strips where appropriate
- maintain operational focus
- avoid giant desktop grids on tablets

6. Responsive behavior
Implement explicit breakpoints:
- mobile
- tablet
- desktop

Do NOT rely solely on generic Tailwind responsiveness.
Design each intentionally.

==================================================

- darker shell
