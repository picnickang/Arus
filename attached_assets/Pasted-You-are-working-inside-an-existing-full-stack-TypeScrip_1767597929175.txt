You are working inside an existing full-stack TypeScript monorepo (frontend + backend) with a very large shared “schema/contract” layer and lots of repeated patterns. Your task is to implement a GOLD-STANDARD deduplication + compaction pipeline and apply safe, high-ROI refactors WITHOUT changing runtime behavior.

HARD CONSTRAINTS
1) Do NOT change any business logic or user-visible behavior.
2) Do NOT rename exported symbols in the public API unless you add backwards-compatible re-exports.
3) Keep changes incremental, commit-safe, and reversible.
4) Prefer codegen/config extraction over clever abstractions.

GOALS
A) Mechanical dedupe: formatting, linting, imports, unused code detection.
B) Duplicate detection: produce a report of duplicated blocks and a prioritized list.
C) Structural compaction: remove repeated plumbing via shared helpers, config-driven patterns, and (where appropriate) codegen.
D) Add CI enforcement so dedupe doesn’t regress.

PHASE 0 — REPO DISCOVERY (DO THIS FIRST)
- Detect package manager (pnpm/yarn/npm) and workspace structure.
- Detect frontend framework (Next/React/etc) and backend framework.
- Detect DB/ORM (Drizzle/Prisma/TypeORM/Kysely/etc).
- Print a short summary in logs: tools found, workspaces, TypeScript configs.

PHASE 1 — MECHANICAL DEDUPE (AUTOMATED)
1) Install and configure:
   - Prettier (format)
   - ESLint + @typescript-eslint (lint)
   - eslint-plugin-import (import hygiene)
   - knip (unused exports/files)
   - jscpd (duplicate code detector)
   - optional: biome (only if it’s already present; otherwise stick to ESLint+Prettier)
2) Add scripts to the root package.json (and workspace scripts if needed):
   - "format": prettier write across ts/tsx/js/json/md
   - "lint": eslint .
   - "lint:fix": eslint . --fix
   - "typecheck": tsc -p (use existing tsconfigs)
   - "knip": knip
   - "dup": jscpd --reporters "html,console" --output "./reports/jscpd"
3) Add config files (only if not present) with sensible defaults:
   - .prettierrc
   - .eslintrc (or eslint.config.*)
   - knip.json
   - .jscpd.json
4) Run all scripts and fix issues with minimal change:
   - Fix only what the tools identify (no refactors yet).
   - Ensure the repo builds and tests pass.

PHASE 2 — DUPLICATION REPORT + PRIORITIZATION
1) Run jscpd and output:
   - ./reports/jscpd (html report)
2) Create a new markdown doc:
   - docs/dedup-report.md
   Include:
   - Top 20 duplicated regions by lines duplicated
   - File paths
   - Categorize duplicates into: API plumbing, DB/repo patterns, UI components, validation/types, utilities.

PHASE 3 — HIGH-ROI, LOW-RISK COMPACTION REFACTORS (DO THESE)
Apply the following refactor playbook ONLY to the top duplicated categories:

A) API / Backend plumbing dedupe
- Identify repeated patterns in route handlers (pagination, filtering, error mapping, auth/tenant scoping).
- Create small focused helpers:
  - paginate(query, params)
  - withTenant(ctx, fn)
  - ok()/fail()/notFound()/badRequest() response helpers
- Replace repeated blocks with helper calls while keeping behavior identical.
- Add unit tests for helpers (or snapshot tests if that’s what exists).

B) Validation / DTO dedupe
- Identify repeated Zod schemas / DTO mappers.
- Create shared primitives (ID, DateRange, Status enums, pagination schema).
- If frontend shares types, ensure no backend-only imports leak to frontend bundles.

C) UI dedupe
- Extract repeated table+filter+toolbar patterns into a small “ListPage” component or config-driven builder.
- Extract common form field components (date picker, select, autocomplete) to a shared UI kit folder.

D) Codegen (ONLY if there is clear repetition)
- If there are many similar CRUD endpoints or clients, add a minimal generator:
  - Input: a small config file listing entity/table name + allowed operations
  - Output: generated files under src/generated/
- Never hand-edit generated files.
- Add a script "generate" and ensure it’s idempotent.

PHASE 4 — ENFORCEMENT (CI + PRECOMMIT)
1) Add a CI job (GitHub Actions if present; otherwise add a generic ./scripts/ci.sh):
   - format check
   - lint
   - typecheck
   - knip
   - duplication threshold guard (jscpd) — start lenient and tighten later:
     - fail only if new duplication is introduced above current baseline
2) Add pre-commit hooks (husky + lint-staged) IF the repo already uses them; otherwise add them with minimal footprint:
   - run prettier + eslint on staged files

DELIVERABLES

QUALITY BAR
