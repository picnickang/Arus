Only three final polish notes (all minor, all worth it):

1) startIngestion() should not block server boot

Right now your pseudocode does await runSqliteBridge(config); which is an infinite loop. If you await it in server/index.ts, your server may never finish starting (routes/UI won’t come up).

Do this instead:

export function startIngestion() {
  const config = loadBridgeConfig();
  runSqliteBridge(config).catch(err => logger.error("Bridge crashed", { err }));
}

That keeps the bridge alive without blocking the HTTP server.

2) Put pipeline stage counters in the right place

arus_ingest_pipeline_stage_total{stage="agent_sqlite_written"} should be emitted by the agent, not Node.

Two options:
	•	Option A (simple): drop that stage in Node and keep:
	•	bridge_frames_read
	•	bridge_readings_decoded
	•	postgres_batch_committed
	•	Option B (full fidelity): agent exposes a /metrics endpoint or writes counters to heartbeat; Node then surfaces it.

I’d start with Option A.

3) Cursor-safe retention depends on ingest_cursor existing

Your agent reads ingest_cursor.last_id. Make sure:
	•	the bridge creates the cursor table on startup (it already does)
	•	agent handles “table missing” gracefully (assume cursorLastId = max(id) or 0)

Practical default:
	•	if ingest_cursor missing, do no deletes (safest).
