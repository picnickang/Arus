ROLE HIERARCHY AND USER/ROLE SYSTEM

Use a simple operational hierarchy.

Required protected roles:

1. super_admin
2. admin
3. vessel_master
4. chief_engineer
5. supervisor
6. technician
7. crew_member
8. logistics_user
9. safety_officer

Optional future roles should not be prioritized unless already required:

* procurement_user
* maintenance_planner
* auditor
* deck_officer
* engineer

Role meanings:

super_admin

* full system owner
* can manage all users, roles, permissions, dashboards, hubs, vessels, credentials, alarm types, alarms, alert logs, and system configuration
* can decide what each role sees
* can decide which admin users can access which admin hubs
* cannot be deleted, disabled, or locked out accidentally
* has all hub access by default

admin

* operational admin user
* can only access admin hubs granted by Super Admin
* does not automatically have access to Maintenance, System Admin, Crew Management, Logistics, AI Analytics, Safety / Alerts, Procurement, or Reports
* can manage only the hubs/areas explicitly assigned to them

vessel_master

* vessel command authority
* dashboard should show vessel readiness, crew status, vessel alarms, safety/compliance issues, logistics blockers, and critical maintenance blockers for assigned vessel(s)

chief_engineer

* vessel engineering authority
* dashboard should show maintenance overview, work orders, equipment alerts, engineering defects, blocked work orders, and engineering safety items for assigned vessel(s)

supervisor

* department/team lead
* dashboard should show assigned team tasks, overdue work, department/vessel issues, approvals, and safety notices for their scope

technician

* hands-on maintenance user
* dashboard should show assigned work orders, personal task feed, relevant equipment alerts, defects they reported, blocked tasks, safety notices, and feedback/flag actions

crew_member

* default regular user
* dashboard should show personal tasks, vessel notices, safety alarms, safety bulletins, acknowledgement items, and feedback/flag actions

logistics_user

* logistics/supply user
* dashboard should show supply requests, pending deliveries, inventory alerts, procurement handoffs if permitted, logistics blockers, and logistics tasks within permitted scope

safety_officer

* safety/compliance user
* dashboard should show active safety alarms, alert log if permitted, safety bulletin acknowledgements, hazard reports, safety flags, and compliance tasks

viewer

* read-only user
* dashboard should show only read-only permitted information
* cannot edit, assign, trigger alarms, update credentials, or change configuration

* regular users cannot access admin APIs
* regular users cannot access role dashboard configuration APIs
* regular users cannot create, edit, delete, or deactivate roles

* fleet-wide safety alarms should be visible to all relevant users
* crew member must not see admin controls
* logistics user should only see logistics data within permitted scope
