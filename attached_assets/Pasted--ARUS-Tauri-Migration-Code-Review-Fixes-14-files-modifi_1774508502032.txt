# ARUS Tauri Migration ‚Äî Code Review Fixes

**14 files modified/created** across 5 severity tiers.

## CRITICAL (fix before any deployment)

### 1. Admin password hashed with bcrypt

**File:** `server/domains/system-admin/routes/auth-routes.ts`

- Passwords now hashed with bcrypt (cost 12) before writing to `.env` as `ADMIN_TOKEN_HASH`
- Verification uses `bcrypt.compare()` instead of `===` string comparison
- Legacy plaintext `ADMIN_TOKEN` auto-migrated to hash on first successful verify
- Password max-length enforced (128 chars) to prevent bcrypt DoS
- `.env` values quoted to handle bcrypt `$2b$` prefix and special characters

### 2. Session tokens hashed before database storage

**File:** `server/domains/system-admin/routes/auth-routes.ts`

- Session tokens SHA-256 hashed before storage in `admin_sessions` table
- Plaintext token returned only to the client; DB never holds raw tokens
- Prevents session hijacking if database is compromised

### 3. Atomic `.env` file writes

**File:** `server/domains/system-admin/routes/auth-routes.ts`

- Write-to-temp-then-rename pattern prevents file corruption on power loss
- Critical for vessel hardware with possible power interruptions

### 4. Setup endpoint restricted to localhost / Tauri origins

**File:** `server/domains/system-admin/routes/auth-routes.ts`

- `POST /api/admin/auth/setup` now rejects requests from non-localhost, non-Tauri origins
- Prevents race condition where an attacker on the vessel LAN sets the admin password
  before the legitimate operator

### 5. Auth status endpoint no longer leaks configuration state

**File:** `server/domains/system-admin/routes/auth-routes.ts`

- `GET /api/admin/auth/status` returns `{ configured: true }` for non-localhost requests
- Only localhost requests see the real configured/unconfigured state
- Prevents attackers from probing whether the setup endpoint is available

### 6. CSP connect-src includes private network ranges

**File:** `src-tauri/tauri.conf.json`

- Added `http://192.168.*:*`, `https://192.168.*:*`, `http://10.*:*`, `https://10.*:*`
- Vessel LAN connections to backends at `192.168.x.x:5000` no longer blocked by CSP

### 7. SQLite admin_sessions table includes user_id and admin_email columns

**File:** `scripts/init-sqlite-schema.js`

- Added `user_id TEXT` and `admin_email TEXT` columns
- Matches what `auth-routes.ts` ‚Üí `storage.createAdminSession()` writes
- Without these, inserts would fail or silently drop fields in vessel mode

## HIGH (fix before production release)

### 8. Updater endpoint fixed for GitHub Releases

**File:** `src-tauri/tauri.conf.json`

- Changed from `https://releases.arus.io/{{target}}/{{arch}}/{{current_version}}`
  to `https://github.com/INSERT_OWNER/INSERT_REPO/releases/latest/download/latest.json`
- Standard Tauri + GitHub Releases pattern that actually works with tauri-action

### 9. `unsafe-eval` added to CSP script-src

**File:** `src-tauri/tauri.conf.json`

- Required by the `new Function('m', 'return import(m)')` dynamic import pattern
- Without this, desktop builds may silently fail to load Tauri plugins

### 10. macOS minimum version bumped to 11.0

**File:** `src-tauri/tauri.conf.json`

- Tauri v2 requires macOS 11+ (Big Sur) for WebKit compatibility
- Was 10.15 (Catalina) which could cause runtime failures

### 11. `getBackendUrl()` fallback returns empty string

**File:** `client/src/lib/desktop.ts`

- Was returning `window.location.origin` which is `tauri://localhost` in desktop mode
- Would cause API calls to `tauri://localhost/api/...` which always fail

### 12. Mid-flow setup persistence fixed

**Files:** `client/src/pages/desktop-setup.tsx`, `client/src/lib/desktopFetch.ts`

- Backend URL saved to localStorage only at wizard completion, not at Step 1
- New `markSetupComplete()` / `isDesktopSetupComplete()` uses dedicated
  `arus-setup-complete` flag instead of checking for backend URL existence
- Prevents wizard skip if app closes between Step 1 and Step 3

### 13. URL validation shared and applied to BackendStep

**Files:** `client/src/lib/urlValidation.ts` (NEW), `client/src/pages/desktop-setup.tsx`,
`client/src/components/admin/DesktopConnectionPanel.tsx`

- Extracted `validateBackendUrl()` to shared utility
- Applied to setup wizard BackendStep (was unvalidated ‚Äî `ftp://invalid` would crash)
- Applied to `testBackendConnection()` as a guard against malformed URLs
