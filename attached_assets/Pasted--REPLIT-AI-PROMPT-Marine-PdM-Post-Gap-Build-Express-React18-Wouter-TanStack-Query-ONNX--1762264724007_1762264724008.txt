2) Profiles + Dataset mixer (the critical ML/data fix)

Implement /apps/server/lib/dataset_mixer.ts that:

Reads feature_windows.len_sec/stride_sec/sampling_hz, baseline_dataset_mix from each profiles/*.json.

Loads sources:

cmapss (NASA) via /apps/server/loaders/cmapss.ts (real URL + mirror & fallback).

marineEngine (Kaggle) via /apps/server/loaders/kaggle.ts (env keys check).

synthetic via /apps/server/loaders/synthetic.ts (deterministic seeded faults).

Combines by weight, windowing to [N, T, F].

Splits train/val; computes thresholds from validation quantile (e.g., 95th).

Store thresholds per profile in checkpoints/<profile>_baseline/thresholds.json.

3) Inference & Explainability

ONNX Runtime: /lib/onnx.ts loads vessel checkpoint if present, else profile baseline; runs sequence reconstruction; outputs MSE + decoded sequence.

Attribution: /lib/attribution.ts computes simple per-feature contribution scores for last window; return top contributors with weights and a short hypothesis string (rule-based mapping per signal).

Adaptive retrain trigger: When confidence < X% for Y hours or KS drift > Z, enqueue a retrain suggestion (logged + visible in /capabilities).
