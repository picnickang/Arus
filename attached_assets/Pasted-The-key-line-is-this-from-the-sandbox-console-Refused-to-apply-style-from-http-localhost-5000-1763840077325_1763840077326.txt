The key line is this from the sandbox console:



Refused to apply style from ‘http://localhost:5000/assets/index-C8omU17M.css’ because its MIME type (‘text/html’) is not a supported stylesheet MIME type


That means:

You are loading http://localhost:5000/ and getting index.html ✅

But when the browser asks for /assets/index-C8omU17M.css, Express is not sending a CSS file.

Instead it’s returning HTML (very likely index.html again via the SPA fallback).



So routing is basically:

Request: / → index.html (fine)

Request: /assets/index-C8omU17M.css

Express hits your catch-all app.get('*') first and sends index.html instead of letting express.static serve the real CSS.

Browser: “This is HTML, not CSS → reject it → blue screen, no styles.”



We just need to make sure static file serving beats the SPA fallback for real files.

Step 1 – Fix the Express routing order



Open your server entry (server/index.ts before bundling) and find where the app is created and static + SPA are configured (this is the part that logs the [Static] Candidate roots messages).



You want something like this exact structure:

// server/index.ts (or wherever your Express app is created)

import express from 'express';
import path from 'path';
import { fileURLToPath } from 'url';

// If you're using ESM:
const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);

const app = express();

// 1) API routes FIRST
// (whatever your API router is called)
app.use('/api', apiRouter); // adjust this line name if different

// 2) Static files from the Vite build
const staticRoot = path.join(__dirname, '..', 'dist');

app.use(
  express.static(staticRoot, {
    index: 'index.html',
  }),
);

/**
 * 3) SPA fallback – ONLY for routes that:
 *    - are NOT /api/*
 *    - do NOT look like file requests (no extension)
 */
app.get('*', (req, res, next) => {
  // Let /api/* and file requests like /assets/foo.css, /assets/app.js fall through
  if (
    req.path.startsWith('/api/') ||
    path.extname(req.path) // has an extension => probably a real file
  ) {
    return next();
  }

  res.sendFile(path.join(staticRoot, 'index.html'));
});

/**
 * 4) Final 404 handler
 *    (for real misses like /api/does-not-exist or missing files)
 */
app.use((req, res) => {
  res.status(404).json({
    error: 'Not Found',
    path: req.path,
  });
});
Important bits:

app.use(express.static(...)) must come before app.get('*', ...).

The SPA fallback only runs when path.extname(req.path) is empty – i.e., URLs like /, /dashboard, /equipment/123, but not /assets/index-C8omU17M.css, /icon-192.png, etc.

The final app.use((req, res) => ...) 404 should be last and must not send that old "Cannot GET /" string.



If you see any code like this anywhere:

app.get('*', (req, res) => {
  res.status(404).send('Cannot GET ' + req.path);
});
Delete or convert it to the JSON 404 handler above and keep it after the SPA fallback.

You want to see:

For /:

HTTP/1.1 200 OK

Content-Type: text/html; charset=utf-8

For /assets/index-C8omU17M.css:

HTTP/1.1 200 OK

Content-Type: text/css; charset=utf-8 (or at least text/css)



If /assets/index-C8omU17M.css still shows Content-Type: text/html, the SPA fallback is still catching it. That means either:

express.static is below your catch-all, or

your app.get('*', ...) doesn’t have the path.extname(req.path) guard.
