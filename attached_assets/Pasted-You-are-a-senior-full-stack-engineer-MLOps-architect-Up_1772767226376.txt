NON-NEGOTIABLE ARCHITECTURE (HEXAGONAL)
========================
- Domain layer: pure TypeScript, no framework imports, no DB/HTTP/logging side effects.
- Ports: interfaces for storage, telemetry source, feature computation, model registry, inference runner, monitoring.
- Application layer: use-cases/services orchestrating ports; owns validation, transactions, and error mapping.
- Adapters: Drizzle DB adapters, Express controllers/routes, background job adapters, model runner adapters, telemetry adapters.
- Routes/controllers must be thin: validate input, call application service, return DTOs.
- All org-scoped operations MUST require orgId and enforce it in storage adapters.
- Keep modules small, cohesive, and testable.
