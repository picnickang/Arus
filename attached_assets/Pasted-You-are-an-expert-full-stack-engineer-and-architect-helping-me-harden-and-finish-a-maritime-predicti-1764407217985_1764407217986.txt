## PHASE 2 – SECURITY & TENANT ISOLATION

Review and, if needed, improve:

1. **API protection**
- Ensure all sensitive routes enforce:
- Auth (if present)
- org/vessel scoping via headers or session (e.g. `x-org-id`, `vesselId`)
- Confirm multi-tenant isolation: no cross-org data leakage.

2. **Secrets management**
- Confirm OpenAI / Turso / PostgreSQL / SMTP / MQTT / StormGeo / Aquametro FMCC credentials are **not**:
- Hard-coded in code
- Sent to the frontend
- Logged in plaintext

3. **Electron security**
- Check BrowserWindow creation:
- `nodeIntegration` disabled in renderers that load remote/local web content
- `contextIsolation` enabled
- Preload scripts minimal and safe
- Content Security Policy:
- Relaxed for dev, locked down for prod

4. Implement only **targeted** fixes (small PR-style changes) where there are clear issues, and summarize them.
