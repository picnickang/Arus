Replit Prompt (paste as-is)

You are a senior desktop architect. I’m building a Windows-first predictive maintenance (PdM) application for vessel/engine telemetry (CAN/J1939/J1587/serial/Modbus possible) that must be reliable offline and performant. I need you to determine which desktop stack is best: Electron, Tauri, .NET (WPF/WinUI), or a hybrid approach (native service/agent + UI shell).

1) Ask me only these inputs (max 10 questions total)
	•	Telemetry source(s): CAN/J1939/J1587/serial/Modbus/vendor gateway/CSV import?
	•	Typical throughput (messages/sec) and peak throughput
	•	Latency/jitter needs: do I need near-real-time UI or can UI update at 1–2s intervals?
	•	Offline requirements: how many hours/days of buffer without loss?
	•	Hardware access: read-only vs control commands (writeback)?
	•	Deployment: single installer? auto-update? admin rights available?
	•	Target devices: rugged PC / shipboard mini-PC / laptop specs (RAM/CPU)
	•	Cross-platform requirement: Windows only now? macOS/Linux later?
	•	Security constraints: code signing, sandboxing, audit logs?
	•	Team constraints: strongest skills (React/Node vs C#/.NET vs Rust/C++)

2) Produce a decision matrix

Make a table scoring each option (Electron, Tauri, .NET, Hybrid) 1–10 on:
	•	Hardware integration ease
	•	Performance (CPU/RAM/startup)
	•	Reliability (offline / long-running stability)
	•	Developer velocity
	•	Security hardening
	•	Packaging + updates
	•	Long-term maintainability

Explain scores briefly and highlight “gotchas” (e.g., Electron multiple processes, native driver needs, WebView limitations, update signing).

3) Recommend ONE architecture

Pick the best approach for my PdM app and justify in 5–10 bullets.

4) Provide a concrete implementation plan

Give:
	•	Process layout (UI app + background agent/service)
	•	IPC choice (gRPC vs WebSocket vs Named Pipes) and why
	•	Storage choice (SQLite, local time-series approach) and data retention strategy
	•	Data flow: ingest → buffer → features → inference → alerts → UI
	•	Update strategy (agent + UI updates)
	•	Minimal MVP milestones (Week 1–4)

5) Include a “fallback plan”

If the chosen stack fails (e.g., WebView issues, driver access), propose the next best pivot with minimal rewrite.

Constraints:
	•	Prefer Windows-first but don’t block future cross-platform
	•	Must handle intermittent connectivity and never lose telemetry if possible
	•	UI is dashboards, alerts, reports; heavy compute/inference can be background
	•	I want the recommendation optimized for real-world shipboard environments (limited resources, long uptime, safety)

Output format:
	•	First: the 10 questions
	•	Second: decision matrix table
	•	Third: final recommendation + plan
	•	Fourth: fallback plan
