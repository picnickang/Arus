	1.	Phase A — E2E Verification

	•	Cursor never advances on forced PG failure (test proves it)

	•	API calls/min reduced (no poll storms)
	•	Renderer memory stabilized (no growth under 10 min idle + dashboard use)
	•	Slowest route mount time reduced

One key dependency to call out explicitly

Performance testing requires:
	•	TelemetryBatchWriter.writeBatch() is bulk insert (multi-values or COPY)

Add one “anti-regression” umbrella

A single CI command should run:
	•	Phase A static audit tests
	•	Phase A core correctness tests
	•	A lightweight perf smoke test (short run, low fps)

That way you never reintroduce legacy ingestion or direct inserts.
