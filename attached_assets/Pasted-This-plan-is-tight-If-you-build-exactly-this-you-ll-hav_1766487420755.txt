The critical piece: TelemetryBatchWriter.flushOnce() contract

You want: cursor advances only after PG commit. That requires flushOnce() to resolve only when the in-flight transaction commit has completed.

Here’s the cleanest way to do it without redesigning your writer:

Minimal approach (Promise queue of “flush waiters”)

Add these fields to TelemetryBatchWriter:
	•	private flushWaiters: Array<{resolve, reject}> = []
	•	private flushing: boolean = false

Add this method:

async flushOnce(): Promise<void> {
  return new Promise((resolve, reject) => {
    this.flushWaiters.push({ resolve, reject });
    this.kickFlush(); // triggers immediate flush cycle if not already flushing
  });
}

Then in your existing flush routine (where you already do PG transaction + commit):
	•	On successful commit: resolve all waiters.
	•	On failure/rollback: reject all waiters.
	•	Always clear waiters.

Pseudo-hook to place at the end of your “commit success” path:

const waiters = this.flushWaiters.splice(0);
waiters.forEach(w => w.resolve());

And in the error path:

const waiters = this.flushWaiters.splice(0);
waiters.forEach(w => w.reject(err));

This is non-invasive and gives the bridge a hard “commit barrier”.
