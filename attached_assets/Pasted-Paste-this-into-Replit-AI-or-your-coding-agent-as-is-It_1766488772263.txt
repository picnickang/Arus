Replit Prompt: Enforce New Telemetry Dataflow End-to-End

You are a staff engineer auditing and patching the ARUS repo to guarantee all telemetry ingestion uses the new system:

C# Agent → SQLite(raw_frames) → Node SQLite Bridge → TelemetryBatchWriter → Postgres

Legacy collectors must be removed or hard-disabled. Your job is to verify dataflow correctness, eliminate bypasses, and add automated checks so regression is impossible.

0) Non-negotiable constraints
	1.	Single ingestion path: there must be exactly one path that writes telemetry into Postgres: TelemetryBatchWriter.
	2.	No direct hardware ingest in Node: no CAN/J1939/serial/MQTT hardware collectors in Node may run or write to Postgres.
	3.	Bridge is the only producer of TelemetryReading in runtime, aside from test fixtures.
	4.	No double ingestion: ensure old processes can’t start (even accidentally).
	5.	Production proof: add runtime asserts + tests verifying the pipeline.

Step 2 — Enforce the new dataflow

Implement the following enforcement mechanisms:

A) Centralize ingestion startup

There must be exactly one runtime startup function for ingestion, e.g.:
	•	server/ingestion/startIngestion.ts

It should:
	•	start the SQLite bridge
	•	start agent status polling (optional)
	•	NOT start any legacy collectors

Update the app bootstrap to call startIngestion() and remove any other collector starts.

B) Hard-disable legacy modules

If legacy collector files still exist, they must be impossible to run:
	•	remove their exports OR throw immediately at module load with a clear message
	•	remove their imports from runtime codepaths
	•	remove scripts that execute them

C) Restrict TelemetryBatchWriter usage

Enforce that only the bridge can call the writer in production:
	•	Create a “guard” utility inside TelemetryBatchWriter:
	•	In production runtime, validate caller context via an explicit “ingestion source tag” argument.
	•	Example: writer.writeBatch(readings, { source: "sqlite-bridge" })
	•	Reject/throw if source != “sqlite-bridge” (except tests).

This prevents accidental reintroduction of direct ingestion.

D) Add “dataflow tracing” markers

Add these markers so you can prove the flow:
	•	In SQLite raw_frames: ensure payload_format_version exists.
	•	In Postgres writes: ensure an idempotency_key includes raw:<source>:<protocol>:<rawFrameId>.
	•	Add a Prometheus counter arus_ingest_pipeline_stage_total{stage=...} incremented at:
	•	agent_sqlite_written
	•	bridge_frames_read
	•	bridge_readings_decoded
	•	postgres_batch_committed
