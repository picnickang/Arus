1. Add a guaranteed root route + SPA fallback



Open your server entry (the one that esbuild bundles to server/index.js – from your logs it’s server/index.ts).



Find the section where Express app is created and where static files are configured (the code that logs [Static] Candidate roots etc).



Replace that whole “static + SPA + error handler” block with something like this:

// server/index.ts (or wherever app is defined)

import path from 'path';
import express from 'express';

const app = express();

// ... all your other imports and middleware

// Make sure staticRoot is exactly the folder the logs show:
// /Users/homeimac/Downloads/RecipeRealm/dist
const staticRoot = path.join(__dirname, '..', 'dist');

// 1) API routes FIRST
app.use('/api', apiRouter); // whatever your API router is called

// 2) Serve static files from root
app.use(express.static(staticRoot));

/**
 * 3) Root route for sanity
 *    This guarantees that GET / sends index.html
 */
app.get('/', (req, res) => {
  res.sendFile(path.join(staticRoot, 'index.html'));
});

/**
 * 4) SPA fallback for anything that is not /api/*
 *    (keeps React Router working on deep links)
 */
app.get('*', (req, res, next) => {
  if (req.path.startsWith('/api/')) {
    return next(); // let API 404 handler deal with it
  }
  res.sendFile(path.join(staticRoot, 'index.html'));
});

/**
 * 5) FINAL 404 handler (for real misses like /api/does-not-exist)
 */
app.use((req, res) => {
  res.status(404).json({
    error: 'Not Found',
    path: req.path,
  });
});
Important:

Make sure there is no other earlier handler that does:

res.status(404).send('Cannot GET ' + req.path);
or similar. If you see one, delete or change it to the JSON handler above and keep it after the SPA fallback.

There should be only one app.get('*', …) SPA route and only one final 404 handler.
