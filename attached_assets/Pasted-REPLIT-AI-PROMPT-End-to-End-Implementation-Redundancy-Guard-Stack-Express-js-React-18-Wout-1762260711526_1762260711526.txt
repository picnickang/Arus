────────────────────────────────────────────────────────
A) AUDIT & REDUNDANCY GUARD (MANDATORY, RUN FIRST)
────────────────────────────────────────────────────────
1) Scan server/routes/** and client/src/**:
- Identify existing endpoints, pages, and components overlapping with F1–F10.
- Detect charts already present (e.g., fuel vs time), context logic, alerts lists, maintenance pages, etc.
- If a collision is found (same route/page/endpoint/component purpose), DO NOT overwrite. Instead:
• Create a proposed replacement under *_v2 or /v2 route
• Benchmark with “Redundancy & Improvement Score” (see below)
• Only switch default routes if the v2 score ≥ current score + 2.

2) Create docs/review/redundancy_report.md listing:
| Area | Existing | Proposed | Reason to Replace or Keep | Score(current→proposed) |

3) Implement a scoring script scripts/redundancy_score.js that evaluates proposed vs existing:
- Criteria (1–5 each): UX clarity, Performance (render time / bundle size delta), Feature coverage, Maintainability (lines/complexity), Consistency (design/system).
- Net Score = sum(proposed) − sum(existing).
- If Net Score < 2 → keep existing; mount proposed under /experimental only.
