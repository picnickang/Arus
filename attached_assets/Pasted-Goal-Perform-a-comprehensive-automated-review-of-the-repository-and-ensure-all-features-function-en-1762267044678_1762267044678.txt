Goal

Perform a comprehensive automated review of the repository and ensure all features function end-to-end. Produce fixes or stubs where missing. Generate a single audit report with green/red status, risk notes, and exact remediation steps.

0) Workspace prep

Detect monorepo layout (/apps/server, /apps/client, /python, /scripts, /docs) and package managers.

Install deps, build TypeScript where present.

Load .env and validate required vars (DATA_DIR, CHECKPOINT_DIR, JWT_SECRET, ADMIN_TOKEN, optional KAGGLE_*).

1) Repository & Architecture Inventory

Build a dependency/ownership map:

Back end: routes → controllers → services → libs; worker/queue usage; config graph (who reads env).

Front end: routes (Wouter), data hooks (TanStack Query), components, charts, map, PWA worker.

Data/ML: dataset loaders, mixer, profiles, training/finetune scripts, registry, lineage/provenance.

Emit /docs/audit/architecture_map.md with:

Module graph (ascii or mermaid), key entry points, cross-cutting concerns, and dead code candidates.

List all public endpoints with method/URL and handler file.

2) Interface Contract Verification (API ↔ UI)

Parse client calls (TanStack Query keys, fetchers) and statically link to server routes.

Generate a contract matrix /docs/audit/api_contract_matrix.md:

For each client call: method, URL, request schema, response schema, exists? (✅/❌), schema match? (✅/⚠️/❌).

For schema mismatches, author zod validators & response mappers or fix types.

3) Feature Parity Checklist



Verify these features exist and are reachable from the UI, with working data paths:

Fleet Overview (RAG cards, filters, mini-map, freshness bars)

Vessel Performance with 4 VPS charts + baseline/fleet overlays + anomaly shading/tooltip (contributors, confidence)

Alerts → Work Orders (ack, assign, close; timeline)

Context Timeline (modes, sea-state chips/events)

Governance Dashboard (Model Lineage browser, Prediction Audit log, Chain verify, exports)

CII (tile, trend, reports export)

Co-Pilot assistant (renders and can call backend)

Saved Views / Preferences

PWA offline (core pages load offline; deferred mutations replay)

Produce /docs/audit/feature_parity.md with ✅/❌ per feature and deep links to code.



If anything is missing, implement it or scaffold it with working endpoints, guarded by feature flags.

4) Data Layer & Database Audit

Auto-detect DB(s) (Postgres/Mongo/SQLite/none). If none, list JSONL/FS stores (lineage, provenance, registry).

Verify:

Connection health, migrations up-to-date, indexes on hot queries (alerts, lineage, provenance).

Data retention flags; PII redaction where applicable.

Create /scripts/db_sanity.ts to run:

Connectivity, migration check, index advice, row counts for hot tables, slow query detection.

Emit /docs/audit/data_layer.md with findings + SQL/NoSQL index suggestions.

If using JSONL/FS stores: check file rolls/rotation, locking, and implement atomic append helpers if missing.

5) ML/Governance Validation

Confirm profiles control dataset mixing, windowing, thresholding.

Confirm lineage records append at train/finetune with artifact hashes & metrics.

Confirm provenance chain hashing on predictions/alerts; run a chain verifier.

Validate drift monitors and adaptive retrain suggestion logging.

Emit /docs/audit/ml_governance.md with:

Sample lineage/provenance entries.

Chain verification result (OK/FAIL with position).

Drift events detected in last N hours (if fixtures present).

6) Performance & Reliability

If absent, create /scripts/perf_harness.js that:

Bombards /score and /vessels/:id/performance with realistic payloads/windows.

Reports p50/p95/p99, error rate, throughput.

Exits non-zero if p95 > PERF_P95_MS (default 1500ms).

Add worker-thread pool for scoring if missing; ensure backpressure and batch sizing are applied.

Emit /docs/audit/performance_benchmarks.md with harness results and flamegraph (if available).

Add/verify GitHub Actions CI job to run npm run perf and fail on regression.

7) Security & Compliance

Automated checks:

JWT RBAC on admin/promo/retrain/verify routes.

zod/joi validation for every POST/PUT/PATCH.

Rate limiting on public & admin routes.

CORS allowlist in prod; helmet headers.

Secrets not logged; dotenv only in server.

Dependency scan (npm audit/pnpm audit) → emit report.

Produce /docs/audit/security_checklist.md with OWASP-style checklist and fix PR stubs where needed.

8) Observability

Confirm /metrics exports:

http durations, inference latency/batch, drift/retrain counters, freshness gauges, queue depth, business SLOs.

Add missing metrics and RED/SRE dashboards JSON templates (Grafana) under /docs/dashboards/.

Emit /docs/audit/observability.md with metric coverage table and dashboard links.

9) Testing Coverage

If absent/incomplete, add:

Unit (Vitest): dataset mixer, thresholds, attribution, provenance/Merkle utils.

API (Supertest): auth/validation errors, success cases for all critical routes.

E2E (Playwright): Fleet→Vessel overlays, Alerts→WO flow, Governance verify chain, Offline replay.

Output coverage summary to /docs/audit/test_coverage.md with gaps & TODOs.

10) PWA & Edge

Run Lighthouse (headless) for PWA checks; confirm offline routes cache and deferred mutations replay.

If edge Dockerfile/Sync agent missing, scaffold minimal versions and document how to run on-vessel.

Emit /docs/audit/pwa_edge.md.

11) Reporting & Exit Criteria

Generate a top-level executive report /docs/audit/audit_report.md that includes:

❇️ Overall Pass/Fail per category (Features, API/UI Contract, Data/DB, ML/Governance, Security, Performance, Observability, PWA/Edge, Tests).

✅ What’s green and why.

🟧 What’s amber (risk) with exact remediation steps & code pointers.

🔴 What’s red (blocking) with proposed diffs/stubs included in PR.

Command block to reproduce every check locally.

Print a terminal summary showing:

p50/p95/p99 for /score & /vessels/:id/performance

Number of endpoints audited & schema matches

Chain verification status

Test pass count & coverage %

Security audit high/critical vuln count

Create a tracking CSV /docs/audit/remediation_backlog.csv with columns:

priority,category,file,summary,owner_suggested,eta,pr_link

12) Deliverables Replit must produce or modify

New / updated scripts:

scripts/db_sanity.ts, scripts/perf_harness.js

scripts/verify_provenance.ts (if not present), scripts/merkle_daily.js (if used)

CI workflow (.github/workflows/ci.yml) running: lint → test → perf → build

Docs:

/docs/audit/* (all files described above)

/docs/dashboards/*.json (Grafana importable)

Minimal code fixes/stubs to make all checks pass or clearly mark feature flag TODOs.

Acceptance Criteria (fail the audit if any “must” is unmet)

API/UI Contract: 100% of client calls map to real routes and pass schema validation.

Core Features: Fleet, Vessel (4 VPS + overlays + anomalies), Alerts→WO, Governance, CII all load with real data or fixtures.

Lineage/Provenance: Chain verifies OK on fresh run; lineage entries exist for latest models.

Performance: npm run perf → p95 ≤ 1.5s (configurable via PERF_P95_MS).

Security: All admin routes RBAC-guarded; request bodies validated; CORS/helmet/rate-limit enabled in prod builds.

Observability: /metrics exposes counters/histograms listed; dashboards generated.

Tests: Unit + API + E2E pass; coverage summary emitted.

Docs: /docs/audit/audit_report.md present and complete with status & remediation backlog.



At completion, print:

A concise PASS/FAIL line per category

Paths to report files

Commands to reproduce checks

A short “What to do Monday” list (top 5 fixes, each ≤ 2 hours)



End of prompt.
