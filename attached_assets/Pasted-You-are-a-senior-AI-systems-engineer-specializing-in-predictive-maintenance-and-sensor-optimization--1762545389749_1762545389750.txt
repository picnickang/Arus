### Required Tests

1️⃣ **Data Ingestion & Normalization**
- Simulate incoming sensor data payloads (rpm, pressure, temp, vibration).
- Validate preprocessing pipeline: unit conversion, NaN handling, normalization.
- Check that all telemetry entries reach the ML buffer (no dropped points).
- Verify timestamp alignment and rolling window buffering.

2️⃣ **Feature Selection & Ranking**
- Use existing feature importance functions to rank sensors by predictive value.
- Ensure at least 3 metrics per sensor: importance, variance, correlation.
- Validate that sensor optimization routine excludes flatline/noisy sensors automatically.

3️⃣ **Model Evaluation**
- Test each model type: RandomForest, XGBoost, LSTM.
- Evaluate performance on sample data (`/data/test_telemetry.csv` or generated synthetic data).
- Expected:
- R² > 0.85 for regression targets (e.g., RUL prediction)
- Precision/recall > 0.8 for classification (failure prediction)
- Output confusion matrix, feature importance, and model latency (ms).

4️⃣ **Optimization Routine**
- Run full optimization pass (sensor selection + model retrain).
- Ensure model weights update and persisted correctly.
- Confirm optimizer logs improvements to `ai_model_optimizations` table.

5️⃣ **Integration Validation**
- Check endpoints:
- `/api/ai/sensor-optimization/status` → returns current model health
- `/api/ai/sensor-optimization/run` → triggers optimization cycle
- `/api/ai/sensor-optimization/metrics` → returns top 10 sensors and gain %
- Assert 200 responses, proper JSON structure, and no undefined values.

6️⃣ **UI & Analytics Hook**
- Mock front-end request for dashboard `/ai/sensor-optimization`.
- Verify chart data matches backend metrics (sensor name, improvement, timestamp).
- Test refresh cycle and ETag response.

7️⃣ **Resilience & Error Handling**
- Inject noisy/outlier data → verify pipeline handles gracefully.
- Drop 20% of sensor readings → confirm fallback interpolation logic.
- Simulate DB disconnection → ensure optimization retries or logs cleanly.
