========================
STEP 2 — TARGET IDEAL ARCHITECTURE (HEXAGONAL + MODULAR)
========================
Define the target architecture as follows and map to this repo:

BACKEND BOUNDED CONTEXT MODULES:
- telemetry (raw + decoded + ingestion health)
- pdm (alerts, risk queue, health score, evidence, RUL)
- models (training runs, drift, registry)
- maintenance (work orders, maintenance records, parts)
- assets (vessels, equipment hierarchy, tags)

Within each module, define folders:
module/
  domain/        (pure, no I/O)
  application/   (use-cases)
  ports/         (interfaces)
  adapters/      (inbound + outbound)
  infra/         (db schema/index provisioning, config)

HEXAGONAL RULES:
- Domain: no DB, no HTTP, no framework imports.
- Application/use-cases orchestrate via ports.
- Inbound adapters: routes/controllers/MQTT handlers call use-cases.
- Outbound adapters: DB repositories, object storage, queue publishers, metrics.

TELEMETRY “IDEAL FLOW” (for reference):
Edge agent -> Ingestion gateway -> (optional stream/queue) -> Raw store -> Decode worker -> Time-series store -> PdM consumers
Mandatory principles:
- Raw-first (store raw payloads before decoding)
- Async decoding + replayable
- Idempotent ingestion (device seq or event_id hash)
- DLQ for bad payloads
- Last-seen freshness tracking
- UTC timestamps

FRONTEND MODULARITY (hexagonal-ish):
src/
  domain/pdm/* (types + rules)
  application/pdm/* (use-cases, selectors)
  ports/PdMApi.ts (interfaces)
  adapters/api/* (HTTP implementation)
  ui/pages/PdMPage/* (responsive components)

UI TARGET (DESKTOP + MOBILE):
- Desktop: KPI cards row; Risk Queue tabs (table); right-side cards (Telemetry Coverage, Model Health, Maintenance Pipeline); Asset Detail panel with charts/actions.
- Mobile: stacked layout; filters in a sheet; KPI cards scroll/grid; risk queue list; asset detail full-screen route or bottom sheet.
(Implement as one page with responsive breakpoints.)
